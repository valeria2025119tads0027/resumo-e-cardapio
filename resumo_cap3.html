Criando Páginas Web com CSS

3.1 Introdução

O CSS (Cascading Style Sheets) é a linguagem que define a "roupa" do site. Enquanto o HTML cuida da estrutura, o CSS controla a cor, a fonte, a organização das listas e a aparência de tabelas e formulários, tornando as páginas mais atraentes.

3.1.1 O Princípio do CSS

Uma regra CSS diz ao navegador exatamente como exibir um elemento. Ela é dividida em:

Seletor: O que você quer mudar (ex: h1 para títulos).
Declaração: A mudança em si, composta por uma Propriedade (o que mudar, como a cor) e um Valor (a nova característica, como azul).

3.1.2 Incluindo CSS em HTML

Existem três formas principais de colocar CSS no seu site:

Externo (.css): Você cria um arquivo separado só para o CSS. É a melhor forma porque você pode usar o mesmo estilo em várias páginas.
Interno (): As regras ficam no topo da página HTML, dentro da tag <head>.
Embutido (Inline): Você escreve o estilo direto na tag HTML (ex: <p style="color: red">). É pouco recomendado pois dificulta a manutenção.

3.1.3 Seletores

Os seletores permitem escolher os elementos com precisão:

Classe (.nome): Seleciona todos os elementos que têm aquela classe. Útil para aplicar o mesmo estilo em vários lugares.
ID (#nome): Seleciona um único elemento específico (o ID deve ser único na página).
Universal (*): Aplica a regra a absolutamente todos os elementos da página.
Filhos e Descendentes: Você pode selecionar um item que está dentro de outro (ex: body > p seleciona apenas os parágrafos que são filhos diretos do corpo da página).

3.1.4 Cascata e Especificidade

Cascata: Se existirem duas regras iguais, a que foi escrita por último no código é a que vale (como se a última "atropelasse" a anterior).
Especificidade: Seletores mais específicos vencem os genéricos. Por exemplo, uma regra para um ID vale mais que uma regra para uma tag comum.

3.1.5 Herança

A herança no CSS é o mecanismo que define quais propriedades de estilo de um elemento pai são transmitidas automaticamente aos seus filhos, facilitando a estilização de textos e fontes de forma global sem a necessidade de repetir regras. Enquanto propriedades tipográficas como color e font-family são comumente herdadas para garantir consistência visual, propriedades de estrutura e layout, como border, não são replicadas para os descendentes, evitando que o design se torne poluído ou excessivamente carregado com estilos desnecessários em cada subelemento.

3.2 Aplicando Cores e Formatação de Texto

3.2.1 Definindo a Cor do Texto e a Cor de Fundo

RGB: A cor é composta pela mistura de Vermelho (Red), Verde (Green) e Azul (Blue). Cada componente varia de 0 a 255.
Exemplo: rgb(255, 255, 0) resulta em amarelo.
HEX: Utiliza um código de 6 dígitos precedido por #. Os pares de dígitos representam R, G e B em hexadecimal.
Exemplo: #FFFF00 (amarelo). Há também a notação curta de 3 dígitos (ex: #FF0).
Nomes de Cores: Uso de palavras-chave pré-definidas. Existem mais de 140 nomes, desde os básicos (ex: yellow) até exóticos como hotpink ou deepskyblue.
RGBA: Semelhante ao RGB, mas inclui um quarto parâmetro: o valor Alpha. Ele define a opacidade entre 0.0 (totalmente transparente) e 1.0 (totalmente opaco).
Exemplo: rgba(255, 255, 0, 0.5) cria um amarelo com 50% de transparência.
HSL: Baseia-se em Matiz (Hue), Saturação (Saturation) e Luminosidade (Lightness).
Hue: Um ângulo de 0° a 360° em um círculo cromático.
Saturação e Luminosidade: Expressas em porcentagens.
Exemplo: hsl(60, 100%, 50%) para o amarelo.
HSLA: Segue a lógica do HSL, mas adiciona o canal Alpha para controlar a opacidade, assim como no RGBA.
Exemplo: hsla(60, 100%, 50%, 0.5).

3.2.2 Elaboração de Textos

1. Definindo as Fontes

A principal propriedade para definir a tipografia de um texto é a font-family. O requisito básico para que uma fonte seja exibida corretamente é que ela esteja instalada ou disponível no computador do usuário.

serif: Fontes com pequenos ganchos ou traços nas extremidades (serifas), comuns em textos impressos.
sans-serif: Fontes sem ganchos, com extremidades retas. São consideradas mais adequadas para telas, pois são mais fáceis de ler em diversas resoluções devido à menor quantidade de detalhes.
monospace: Fontes não proporcionais, onde todos os caracteres têm a mesma largura. É a escolha ideal para exibir códigos-fonte.
cursive: Fontes que imitam a escrita à mão. Recomendadas apenas para acentos visuais e casos excepcionais, nunca para textos longos.
fantasy: Fontes decorativas. Assim como as cursivas, devem ser usadas com moderação e não para blocos de texto contínuos.

2. Ajustando o Tamanho das Fontes

A propriedade font-size é utilizada para modificar o tamanho dos textos em uma página web. O material destaca três unidades principais para essa especificação:

Pixel (px): Define o tamanho exato da fonte em píxeis de tela. Por exemplo, 14px significa que a fonte terá exatamente 14 píxeis de altura. É uma unidade de medida absoluta.
Percentagem (%): É uma unidade relativa baseada no tamanho padrão do navegador ou nas configurações do utilizador.
Exemplo: Como a maioria dos navegadores utiliza 16px como padrão, definir uma fonte como 150% resultará num tamanho de 24px ($16 \times 1,5$).
Unidade em: Também é uma unidade relativa baseada na largura da letra "m" (e, por extensão, no tamanho de fonte atual do elemento pai).
Exemplo: Definir um cabeçalho como 1.5em significa que ele terá 1,5 vezes o tamanho da fonte padrão aplicada.

3. Ajustando o Estilo da Fonte

A aparência visual do texto em termos de inclinação e "peso" é controlada por duas propriedades principais: font-style e font-weight.

Estilo da Fonte (font-style):

Esta propriedade define se o texto deve ser exibido inclinado ou normal. Os valores possíveis são:
italic: Exibe o texto em itálico.
oblique: Exibe o texto oblíquo (uma inclinação mecânica da fonte normal).
normal: Define o texto sem inclinação (padrão).

Espessura da Fonte (font-weight)

Esta propriedade define o "peso" ou a grossura dos caracteres. Ela oferece flexibilidade através de palavras-chave ou valores numéricos:

I) Valores de Palavras-chave:

normal: Espessura padrão (equivalente ao valor 400).
bold: Texto em negrito (equivalente ao valor 700).
lighter / bolder: Tornam o texto um nível mais fino ou mais grosso em relação ao elemento pai.

II) Valores Numéricos:

Permite gradações finas entre 100 e 900 (em centenas).
Quanto maior o número, mais grossa é a fonte. Por exemplo, uma fonte com font-weight: 900 será extremamente pesada/grossa, enquanto 100 será muito fina.

4. Outras Opções de Formatação de Fontes

Além de formatar a fonte, o tamanho da fonte e o estilo da fonte, o CSS oferece muitas outras maneiras de formatar fontes:

I) Transformação e Decoração de Texto

text-transform: Altera a caixa das letras. Pode transformar o texto todo em maiúsculas (uppercase), minúsculas (lowercase) ou capitalizar a primeira letra de cada palavra (capitalize).
text-decoration: Adiciona linhas decorativas ao texto, como sublinhado (underline), tachado/riscado (line-through) ou linha superior (overline).

II) Espaçamentos

line-height: Controla o espaçamento vertical entre as linhas de um texto.
letter-spacing: Controla o espaço horizontal entre letras individuais.
word-spacing: Controla o espaço horizontal entre as palavras.

III) Alinhamento e Recuo

text-align: Alinhamento horizontal do texto. Valores comuns: left (esquerda), right (direita), center (centralizado) e justify (justificado).
vertical-align: Alinhamento vertical do texto em relação ao seu contêiner. Valores: top (topo), bottom (base) e middle (meio).
text-indent: Cria um recuo (espaço em branco) especificamente na primeira linha de um parágrafo.

IV) Efeitos Visuais

text-shadow: Aplica efeitos de sombreamento diretamente nos caracteres do texto, conferindo profundidade.

5. Formatação de Links

Os links podem ser estilizados de forma diferente dependendo do estado em que se encontram.

:link - um link normal, não visitado.
:visited - um link que o usuário visitou.
:hover - um link que aparece quando o usuário passa o mouse sobre ele.
:active - um link no momento em que é clicado
:focus - permite a formatação de links quando eles recebem o foco.

3.3 Listas e Tabelas

Componentes estruturais como listas e tabelas podem ter suas aparências completamente customizadas através do CSS. Esta seção introduz as propriedades específicas disponíveis para manipular esses elementos.

3.3.1 Design de Listas

A principal forma de personalizar listas no CSS é modificando os marcadores (bullets) que antecedem cada item da lista. A propriedade utilizada para essa finalidade é a list-style-type.
Os valores aceitos por essa propriedade variam dependendo se a lista é do tipo ordenada (<ol>) ou não ordenada (<ul>).

1. Formatação de Listas Não Ordenadas

Listas não ordenadas são aquelas em que os itens não possuem uma sequência numérica ou alfabética rígida. Por padrão, os navegadores exibem cada item precedido por um ponto preto sólido. Com a propriedade list-style-type, é possível alterar esse comportamento usando os seguintes valores:

none: Remove completamente qualquer marcador da lista (útil para criar menus de navegação).
disc: Aplica o marcador padrão, que consiste em um círculo preto preenchido.
circle: Altera o marcador para um círculo vazado (branco por dentro com uma borda preta).
square: Transforma o marcador em um quadrado preto sólido.

2. Formatação de Listas Ordenadas

Listas ordenadas (<ol>) são aquelas em que os itens seguem uma sequência lógica ou classificação. Em vez de símbolos geométricos, elas utilizam sistemas numéricos ou alfabéticos.
A propriedade list-style-type permite controlar e alterar o formato dessa ordenação através de diversos valores, incluindo:

decimal: É a numeração padrão. Exibe os itens utilizando algarismos arábicos normais (1, 2, 3, ...).
decimal-leading-zero: Semelhante ao decimal, mas adiciona um zero à esquerda para números de um único dígito (01, 02, 03, ... 09, 10, 11, ...).
lower-alpha: Utiliza letras minúsculas do alfabeto para enumerar a lista (a, b, c, ...).
upper-alpha: Utiliza letras maiúsculas do alfabeto para a ordenação (A, B, C, ...).
lower-roman: Realiza a contagem utilizando algarismos romanos em letras minúsculas (i, ii, iii, iv, ...).
upper-roman: Realiza a contagem utilizando algarismos romanos em letras maiúsculas (I, II, III, IV, ...).

3.4 Sistemas de Layout

Os sistemas de layout em CSS determinam como os elementos HTML são organizados e posicionados na tela do navegador. O desenvolvimento web evoluiu de métodos baseados em tabelas e truques de posicionamento para sistemas nativos modernos e altamente robustos.

1. Float

A propriedade float define o posicionamento de elementos empurrando-os para as extremidades laterais do seu elemento pai.

Como funciona: Ao aplicar float: left; ou float: right;, o elemento é deslocado para a esquerda ou para a direita, saindo parcialmente do fluxo normal da página. Isto faz com que os elementos seguintes (como blocos de texto) subam e contornem o elemento flutuante.

2. Flexbox

O Flexbox é um sistema de layout unidimensional, projetado para alinhar e distribuir espaço entre elementos numa única dimensão de cada vez — ou numa linha horizontal ou numa coluna vertical.

Como funciona: O modelo é ativado aplicando display: flex; no elemento pai (o Flex Container). A partir desse momento, todos os elementos filhos diretos (Flex Items) ganham um comportamento flexível e ajustam-se automaticamente ao espaço disponível.
Eixos de alinhamento: Trabalha com o Eixo Principal (Main Axis), que dita a direção dos itens via flex-direction (row para horizontal ou column para vertical), e com o Eixo Cruzado (Cross Axis), que corre de forma perpendicular ao principal.

Propriedades essenciais:

justify-content: Distribui o espaço extra e alinha os itens ao longo do Eixo Principal (centralizado, espaçamento igual, etc.).
align-items: Controla o alinhamento vertical/horizontal dos itens ao longo do Eixo Cruzado.
flex-grow e flex-shrink: Permitem que os elementos aumentem para preencher espaços vazios ou encolham para evitar que o layout quebre.

3. Grid

O Grid Layout é um sistema de posicionamento bidimensional. Ele permite gerir e alinhar elementos em duas dimensões em simultâneo — controlando linhas (horizontal) e colunas (vertical) ao mesmo tempo.

Como funciona: É ativado aplicando display: grid; no elemento pai (o Grid Container). Em vez de depender do tamanho individual dos filhos para ditar o layout, define-se primeiro uma malha rígida (grade) no elemento pai e, em seguida, posicionam-se os filhos nas células dessa estrutura.

Propriedades essenciais:

grid-template-columns e grid-template-rows: Permitem desenhar explicitamente o número e o tamanho das colunas e das linhas da página.
gap (ou grid-gap): Define o espaçamento de canaleta (linhas em branco que separam as células) sem a necessidade de aplicar margens complexas nos elementos filhos.
Unidade fr (Fraction): Unidade de medida exclusiva do Grid que representa uma fração do espaço restante disponível (ex: 1fr 2fr divide o espaço em duas colunas, onde a segunda tem o dobro do tamanho da primeira).
